# Judge results: reproduce in one command

Every external headline number regenerates from the committed `../../data/judgments.jsonl` and the committed length table in this folder. No GPU, no model download, no external data.

```bash
python ../../src/analyze_judgments.py --judgments ../../data/judgments.jsonl
```

Expected output (matches `robustness_summary.json` in this folder):

- Win rate: PaRLA 83.8% / base 6.6% / tie 9.6% (419 / 33 / 48 of 500)
- Sign test on decided cases: z = 18.1 (p < 1e-50)
- Major omissions per report: base 3.99, PaRLA 1.36
- Hallucination: base 12.6% of cases, PaRLA 14.0% (67 vs 85 items)
- Length control: PaRLA longer in 83.2% of cases (1.39x); on the 84 cases where PaRLA is not longer, it still wins 56%

Survival significance (from `../survival/survival_fold_results_long.csv`): the pooled paired test across all 25 fold-pairs is significant (mean +3.2 points, t(24) = 2.27, p ≈ 0.03, 17/25 folds favor PaRLA); no single cohort is individually significant at 5 folds.

## Files

- `figure_win_rates.csv`: win counts and rates.
- `figure_category_scores.csv`: per-criterion means, 95% CIs, and deltas.
- `robustness_summary.json`: the numbers above, regenerated by `analyze_judgments.py`.
- `generation_lengths.csv`: per-report base/PaRLA output lengths (IDs and character counts only), reproducing the length control without the raw generation shards.
